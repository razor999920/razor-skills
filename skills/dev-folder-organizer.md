# dev-folder-organizer

**Group:** Personal & workflow

## What it does
Sorts and files projects in `~/Documents/Development/Projects` into domain-based category folders: `apps/`, `games/`, `tools/`, `web/`, and `learning/`. It classifies each project by **reading the project's own docs** — `README.md` first, then `CLAUDE.md`, then `AGENTS.md` — and only inspects raw files if no docs exist.

## Key rules
- Structure stays **flat**: one level only, `category/project`.
- Applies naming hygiene.
- **Never deletes** a project without explicit confirmation.

## Triggers
"organize / sort / tidy / clean up / reorganize the dev folder", "file" or "put away" a project, "where does this project go", or when a new project needs a home.

## Notes
This is the skill behind keeping your Projects directory coherent — including this `razor-skills` repo, which would file under `tools/` or `learning/`.
